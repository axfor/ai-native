# 统一认证中心 — 架构验证文档

> 本文档用于验证架构设计的合理性，覆盖安全性、性能、可维护性、合规性四个维度。
> **培训演示时**：可将本文档作为 Prompt 喂给 AI，让 AI 基于架构文档进行评审。

## 一、架构评审检查清单

### 1.1 安全性验证

| # | 检查项 | 预期 | 验证方法 |
|---|--------|------|---------|
| S-01 | 密码存储使用 bcrypt ≥ 12 轮 | ✅ | 检查 auth.service.js 中 bcrypt.hash 的 saltRounds |
| S-02 | JWT 使用非对称签名 (RS256) | ✅ | 检查 token.service.js 的签名算法 |
| S-03 | Access Token 过期时间 ≤ 15 分钟 | ✅ | 检查 config 中的 JWT_ACCESS_EXPIRES |
| S-04 | Refresh Token 过期时间 ≤ 7 天 | ✅ | 检查 config 中的 JWT_REFRESH_EXPIRES |
| S-05 | 私钥不在代码仓库中 | ✅ | 检查 .gitignore 包含 keys/private.pem |
| S-06 | 密码不出现在 API 响应中 | ✅ | 检查 User 序列化逻辑排除 passwordHash |
| S-07 | 密码不出现在日志中 | ✅ | 检查审计日志是否脱敏 |
| S-08 | SQL 注入防护 | ✅ | Prisma ORM 参数化查询 |
| S-09 | 登录限流 ≤ 5 次/分钟 | ✅ | 检查 rate-limit 中间件配置 |
| S-10 | HMAC 时间窗口 ≤ 5 分钟 | ✅ | 检查 apikey.service.js 的时间校验 |
| S-11 | TOTP Secret 加密存储 | ✅ | 检查数据库中 mfaSecret 是否加密 |
| S-12 | OAuth2 state 参数防 CSRF | ✅ | 检查 OAuth 流程中 state 的生成和校验 |

### 1.2 性能验证

| # | 检查项 | 预期 | 验证方法 |
|---|--------|------|---------|
| P-01 | Token 校验不查数据库 | ✅ | JWT 验签是纯计算操作，只查 Redis 黑名单 |
| P-02 | Redis 黑名单查询 < 1ms | ✅ | Redis GET 操作 |
| P-03 | 黑名单 Token 自动过期 | ✅ | Redis TTL = Token 剩余有效期 |
| P-04 | 密码校验 < 300ms | ✅ | bcrypt 12 轮约 200-300ms |
| P-05 | 数据库查询有索引 | ✅ | username, email, oauth(provider+providerId) 有唯一索引 |

### 1.3 可维护性验证

| # | 检查项 | 预期 | 验证方法 |
|---|--------|------|---------|
| M-01 | 认证策略可插拔 | ✅ | Passport.js 策略模式，新增认证方式只需添加策略 |
| M-02 | 配置外部化 | ✅ | 所有密钥和配置通过环境变量注入 |
| M-03 | 错误处理统一 | ✅ | 全局 error-handler 中间件 |
| M-04 | 业务逻辑与路由分离 | ✅ | Routes → Services → Data 三层架构 |
| M-05 | 数据库迁移可控 | ✅ | Prisma Migration 管理 |

### 1.4 合规性验证

| # | 检查项 | 预期 | 验证方法 |
|---|--------|------|---------|
| C-01 | 审计日志记录关键操作 | ✅ | login/logout/password_change/mfa_change 均有日志 |
| C-02 | 支持账号删除（GDPR） | ✅ | 用户可请求删除账号和关联数据 |
| C-03 | Token 不含敏感信息 | ✅ | JWT Payload 只含 userId, role, iat, exp |

## 二、架构风险点与缓解措施

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Redis 宕机导致黑名单失效 | 高 | 低 | 降级为内存 Map + 日志告警 |
| JWT 私钥泄露 | 极高 | 极低 | 密钥轮换机制（kid 字段）、HSM 保护 |
| OAuth 提供商服务中断 | 中 | 低 | 本地密码作为 fallback 登录方式 |
| 数据库连接池耗尽 | 高 | 中 | Prisma 连接池配置、限流保护 |
| TOTP 时钟偏移 | 低 | 中 | 允许 ±1 个时间窗口 (30 秒) |

## 四、测试验证矩阵

| 功能模块 | 单元测试 | 集成测试 | 安全测试 |
|----------|---------|---------|---------|
| 用户注册 | 密码强度校验、重复用户名 | 完整注册流程 | SQL 注入、XSS |
| 用户登录 | 密码验证、账户状态检查 | JWT 签发和验证 | 暴力破解限流 |
| Token 刷新 | Refresh Token 校验 | 新 Access Token 签发 | 过期 Token 重用 |
| Token 黑名单 | Redis 操作 | 登出后 Token 失效 | 竞态条件 |
| OAuth2 登录 | state 校验、用户创建 | 完整 OAuth 流程 | CSRF via state |
| MFA/TOTP | TOTP 生成和校验 | 启用→登录→验证流程 | 暴力猜测限流 |
| RBAC | 角色判断逻辑 | 越权访问拒绝 | 权限提升攻击 |
| API Key | HMAC 签名校验 | 完整 API Key 认证 | 重放攻击 |
| 审计日志 | 日志记录正确性 | 关键操作均有日志 | 日志注入 |
